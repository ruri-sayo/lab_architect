# 1.不正システムについて
実装コンセプトとしては、「バレなければOK、ただし逃げ切っても破滅」とします。

|不正回数|内部告発（Lv.2）|年次査察（Lv.1）|10年完走時の結末|
|---|---|---|---|   
|0回|発生しない|安全|ノーマル/グッドエンド|
|1-3回|確率で発生（逮捕リスク）|確率で発生（逮捕リスク）|ライトBADエンド|
|4-9回|確率で発生（逮捕リスク）|確率で発生（逮捕リスク）|バッドエンド（社会復帰不可）|
|10回以上|無効（もみ消し）|発生（確率は最大）|ワーストエンド（公正研究法違反で逮捕・学会除名で歴史から抹殺）|

このような感じで、不正を起こした時点でノーマルエンドは起こしません。
もみ消しシステム: 不正が10回を超えると、研究室が「基幹研究室」として学内に根を張るため、内部告発が機能しなくなります。ただし、外部（査察）の目はより厳しくなり、バレた際のダメージは最大化されます。
「不正は『短期的なブースト』と引き換えに、『完走時の勝利条件』を不可逆的に破壊するフラグである」と定義します。


# 2.ゲームバランス
既出の研究室のランク別予算はそのまま維持。
戦力不足に対応するため、Sランク研究室には、外部から50%の確率でM1、D1が来る（パラメータはランダム）
ph.Dは研究室の割当資金の5%を年単位で必要。（つまり、ph.Dを受け入れると、予算が95％になる）
ただし、必要に応じて、configをいじることで、ph.Dの超高コストに対応する高性能ユニットにはする。

# 3.UI/UX：居室（ラボ）ビューの採用
スマホの画面制約を解決する「居室切り出し演出」に変更する。

「わちゃわちゃ感」の演出:
画面中央に「学生居室」を表示。
学生が代わる代わるやってきて、「実験成功です！」「器具壊しました...」と吹き出しで報告。

全8人を同時に動かすのではなく、「今、活動報告に来た学生」にスポットを当てることで、小さな画面でもキャラ性が際立ちます。

オートフェーズのスキップ: 事務作業（PL/BSの確認）の裏側で動いている演出にし、忙しい時は「結果サマリー」へ即座に飛べるようにする。

# 4. 勝利条件の変更
新・勝利条件: 「10年間、研究室を潰さずに運営しきること」。

スコアの意義: 「真理」を削除する代わりに、クリア時の「称号（例：地方の雄、学界の重鎮、虚飾の怪物）」を変化させる方向にシフトします。

## 5.3.チュートリアル
### 5.3. チュートリアル
#### 導入フロー
1. **タイトル画面**: 「はじめから」を選択。
2. **導入演出**: サポートキャラ「佐伯 栞里（さえき しおり）」が登場し、挨拶と自己紹介。
3. **研究室命名**: プレイヤーが研究室の名前を決定。
4. **基本説明**: ゲームの進め方、リソース管理、勝利条件の解説。
5. **プレイ開始**: 1年目4月（第1回ラボミーティング）へ。
