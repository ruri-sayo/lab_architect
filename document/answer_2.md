# 1.逮捕・告発時の発生確率について
倫理（不正）システムの確率設計
- 内部告発リスク（毎月判定）:$$P_{Whistle} (\%) = \max(0, 5 + \text{不正回数}) \quad (\text{不正回数} \ge 10 \text{ の時 } 0\%)$$
- 年度査察リスク（毎年4月判定）:$$P_{Audit} (\%) = \min(20, 10 + \text{不正回数})$$
- データ捏造（コマンド）: 実験失敗時に選択可能。獲得Data ptを 20pt に置換し、不正カウンターを +1 する。

# 2. 不正10回以上のもみ消しシステムについて
ペナルティは設けない。
補充ユニットは名前だけ変更した同一個体。

# 3. 称号
称号の合成ロジック
最終的な称号を以下の3パートで構成する：
[不正接頭辞] + [ランク別接頭辞] + [ユーザー入力の研究室名]

不正接頭辞（確定）
0回：(なし)
1〜3回：不正の
4〜9回：虚飾の
10回以上：虚構の

ランク別接頭辞
3年目〜10年目の平均ランクに基づき、いくつか用意した中から抽選。
|ランク|接頭辞|
|---|---|
|S|究極の、至高の、世界的な、神話級の、伝説的な|
|A|名門の、傑出した、次世代を担う、盤石な、優秀な|
|B|新進気鋭の、堅実な、地方の雄、期待の|
|C|発展途上の、無名の、一介の|
|D|未熟な、稚拙な、拙劣な|
|F|未成熟な、見込みはある  |

# 4. データ増強（ブースト）の具体的な挙動
実験失敗時に、「失敗を受け入れる」と「データを捏造する」を選択可能にし、データを捏造すると、20pt分のデータを獲得し、不正カウンターを+1する。

# 5. UI演出：逮捕イベントの表現
- 役割: 最後までプレイヤーを信じて並走する「共犯者」。
- 逮捕演出: 彼女の落胆したセリフ（「信じていたんですよ？」等）の後に、報道風のバッドエンド画面へ遷移。
栞里の発言例：「先生。残念でした。」「栞里、先生のこと信じていたんですよ？」など

# 6. リセットマラソンの防止
- オートセーブ: 毎ターン実行し、不正後のリセマラを抑制。
- 技術的介入: ソース解析やデバッグモードによる改変は制限しない（個人の自由とする）。

SNS画像：寄せ書きシステムのビジュアル定義<br>
