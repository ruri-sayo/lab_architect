# 設計議論・検討事項：ラボ・アーキテクト！ (v1.0)

「基本設計書 v6.0」から具体的な実装設計に落とし込む上で、詳細の詰めや議論が必要な項目をまとめました。

## 1. 倫理（不正）システムの整合性
- **矛盾の解消**: セクション6では「不正カウンターLv.1〜3による段階的なリスク」が語られていますが、セクション9.4では「原則アウト（ゲームオーバー）」とされています。
  - **提案**: 「外れ値除外（QRPs）」程度ならカウンター蓄積による査察リスク、「データ捏造」なら即座に逮捕・追放エンド、といった重みの差をつけるべきか？
- **演出の意義**: 9.4の「成功・得点化しない」という点について。全くメリットがない場合、プレイヤーが選択する動機が失われます。「短期的には爆発的にData ptが増えるが、次の年度更新を越せる確率が致命的に下がる」といったリスクリターン設計が必要ではないか。

## 2. 活動フェーズ（SD演出）の具体化
- **プレイヤーの介入**: オート演出中、プレイヤーは「ただ見るだけ」なのか、あるいは「気合を入れる（リソース消費で一時的に成功率UP）」などの介入要素を持たせるか。
- **情報提示**: 実験中の失敗や器具破損が起きた際、ログとして流すのか、吹き出し等で演出するのかの定義が必要です。

## 3. 「真理（Nature級）」到達の具体条件
- **計算式以上のフラグ**: 単なる累計スコアだけでなく、特定の「☆3：実験的テーマ」で「当たり」を引き、かつ「特殊データ」を一定量保持した状態で「Hard論文」を採択させることが必須条件となるか。
- **周回要素**: 1回のプレイで到達可能なバランスにするのか、あるいは「継承（前回の研究成果を一部引き継ぐ）」などの要素が必要か。

## 4. リソースバランスの数式化
- **交付金の計算式**: スコアに連動するとのことですが、具体的に「いくら」もらえるのかの定義。
  - $予算 = 基本額 + f(前年度スコア)$
- **学生配属の優先順位**: 学生が配属される際、プレイヤーは学生を選べるのか（ガチャ要素）、それとも自動的に配属されるのか。

## 5. 役職とAPのバランス
- **Ph.Dの扱い**: 維持費（研究費5%）に加えてAP 10という設定ですが、中盤以降の主力になる想定か。維持費が「売上の5%」なのか「固定費」なのかで運用難易度が変わります。
- **B4の教育**: 「勉強」アクションがどの程度パラメータを伸ばすのか。卒業までにM、Dへと引き上げるための最適な育成曲線の設計。

## 6. 技術的な制約とUI
- **セーブデータ**: 10年間（120回）という長丁場を考慮し、ローカルストレージへのオートセーブ実装。
- **レスポンシブデザイン**: メインラボの「学生がわちゃわちゃ動く」画面がスマホの縦画面でどこまで視認性を確保できるか。
