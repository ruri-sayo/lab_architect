# 実装スケジュール：ラボ・アーキテクト！ (v1.0)

本ドキュメントは「ラボ・アーキテクト！ 基本設計書 v6.0」に基づき、開発工程をフェーズ分けしたものです。

## フェーズ 1：基礎基盤の構築
- **プロジェクト環境のセットアップ**
  - Vite + React + Vanilla CSS によるモダンな開発環境の構築。
- **ターン管理システムの実装**
  - 120ターン（10年間）のカウントダウンと、各フェーズ（指令・活動・決裁・年度更新）の遷移ロジック。
- **リソース管理ロジック**
  - 資金（圓）、Data pt、特殊データの状態管理。

## フェーズ 2：学生（ユニット）システム
- **ユニットデータモデルの作成**
  - 役職（B4, M, Ph.D）による初期AP・維持費・成長率の違いを実装。
- **パラメータ影響ロジック**
  - Dex, Comp, Diligence が実験成否や成功率に与える影響の計算。
- **教育・作業指示システム**
  - APを消費してパラメータを上げる「勉強」や、各実験への割り当て機能。

## フェーズ 3：研究・設備・メンテナンス
- **研究ティア構造の実装**
  - Tier 1〜3の遷移と、「当たり/ハズレ」の内部フラグ管理。
  - 研究テーマ期待度（☆1〜☆3）による成功率・Tier上限の制御。
- **設備メンテナンスシステム**
  - Tiers 1〜4の購入と、ランダム発生する故障（Dry/Wet）・修理ロジックの実装。

## フェーズ 4：決裁・投稿・年度更新
- **論文投稿システム**
  - Easy, Middle, Hard の選択と、Data pt 投資による採択率変動の計算。
  - 査読結果（Reviewer 2からのコメント含む）の演出。
- **年度更新ロジック**
  - 前年度スコアに基づく予算算出と、新入生配属（0〜8人）の抽選。

## フェーズ 5：倫理・スコアリング・エンディング
- **倫理（不正）システム**
  - インターフェイス上の「悪魔の選択」実装と、不正カウンター/即死判定の統合。
- **スコアリングシステム**
  - 最終スコアの計算と、到達ランクの表示。
- **エンディング分岐**
  - 「真理」への到達、資金ショート敗北、逮捕エンドなどの実装。

## フェーズ 6：全般的な演出とポリッシュ
- **活動フェーズのビジュアル演出**
  - SDキャラのオート演出（ドット絵ライクな挙動）の実装。
- **UI/UXのブラッシュアップ**
  - 秘書室（栞里）との対話、論文採択時の高揚感のある演出。
- **レスポンシブ対応**
  - PCおよびスマートフォン両方での快適な操作性の確保。
