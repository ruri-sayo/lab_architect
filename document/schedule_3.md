# 確定実装スケジュール：ラボ・アーキテクト！ (v3.0)

最終基本設計書 v7.0 に基づき、フェーズ 4, 5 を中心に詳細化した確定ロードマップです。

## フェーズ 1：基礎基盤とチュートリアルの実装
- **プロジェクト環境のセットアップ**
  - Vite + React + Vanilla CSS (module) による開発環境構築。
- **チュートリアル・命名システム**
  - 佐伯栞里による導入会話コンポーネント。
  - 研究室名入力＋接尾辞選択（プルダウン）UI。
  - 初期2年間の「立ち上げ期間」フラグ管理。

## フェーズ 2：改良版「居室ビュー」UIの実装
- **居室ビュー・フォーカス演出**
  - 中央に配置された「学生居室」に、学生が入れ替わり立ち替わり現れるアニメーション。
  - 吹き出し（実験成功、破損報告など）による情報の視覚化。
- **オートフェーズ・高速化**
  - 事務作業の裏で演出を回す仕組みと、即座に「結果サマリー」へ遷移できるスキップ機能。

## フェーズ 3：高等ユニット（M1/D1/Ph.D）とバランス調整
- **外部流入ロジック**
  - Sランク研究室における50%の確率でのM1/D1配属処理。
- **Ph.D 維持費・高性能化**
  - 年間予算の5%を維持費として自動控除。
  - コンフィグベースでの高AP・高成功率調整。

## フェーズ 4：不正・もみ消し・告発システムの精緻化
- **不正カウンターと分岐ロジック**
  - **救済「禊（みそぎ）」の実装**: 不正1-3回で発覚時、翌年度ランクF・学生0・予算激減でリスタートさせる処理。
  - **10回以上の「もみ消し・補充特権」**: 
    - 内部告発の確率を0%に固定。
    - 学生HP=0時に即時補充。**補充ユニットは前任者の全パラメータ+10**された強化個体とする。
  - **年次査察**: 確率上限を20%に設定し、「生かさず殺さず」完走させるバランス調整。

## フェーズ 5：マルチエンドとSNSシェア
- **称号計算エンジンの実装**
  - 不正回数（接頭辞）+ ランク実績 の組み合わせロジック。
- **エンドロール演出**
  - 5パターン（Good, Normal, ライトBad, Bad, Worst）の実装。
- **SNSシェア機能の拡張**
  - クリア時だけでなく、**ゲームオーバー（逮捕・破産）時**にもシェア可能に。
  - 画像生成：不正なし（満員の寄せ書き） / 不正10+（空白・無機質） / 逮捕（破れた色紙）。

## フェーズ 6：最終調整とデバッグ
- **コンフィグ調整機能**
  - バランス調整用のデバッグUI。
- **レスポンシブ最終確認**
  - スマホプレイ時のUX最適化。
