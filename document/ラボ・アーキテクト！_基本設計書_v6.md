ラボ・アーキテクト！ (Lab Architect!) - 最終基本設計書 (v6.0)

1. プロジェクト概要

タイトル: ラボ・アーキテクト！

ジャンル: 研究室運営・背徳経営シミュレーション

プラットフォーム: Webブラウザ（PC/スマホ対応）

舞台設定: 五洲国（ごしゅうこく）。明治〜昭和初期のレトロな美学と近未来技術が融合した技術立国。作用技術大学（STU）医工学系新規研究室。

プレイヤーの目的: 10年間（120回）のターン制限の中で、研究室の資金・名声・成果を最大化し、「真理」へと到達すること。

2. ゲームシステム・コアサイクル

2.1. 基本サイクル（1ヶ月/1ターン）

司令フェーズ (Command): APを消費し学生への指示、備品の購入、設備の修理。

活動フェーズ (Action): SDキャラによるオート演出。実験の成否、サボり、器具破損の発生。

決裁フェーズ (Decision): 実験報告。**「悪魔の選択（データ加工・捏造）」**の実行。

年度更新 (Yearly): 毎年4月に、前年度スコアに基づき予算と新入生配属を決定。

2.2. 研究室評価スコア

$$Score = \alpha \times (\text{累積論文数} + \text{学会発表数}) + \beta \times \text{平均親愛度} + \gamma \times \text{純資産}$$


※このスコアが翌年度の運営費交付金および学生配属数（0人〜最大8人）に直結する。

3. リソースと設備管理

3.1. 資金・備品

圓 (En): 運営資金。人件費、備品、修理費に消費。

データ pt: 論文投稿のためのリソース。

特殊データ: Tier 3当たりルートのみで得られる希少リソース。Hard論文に必須。

3.2. 設備メンテナンス

実験により確率（学生の真面目さに依存）で器具が破損する。

Dry系破損: 獲得データ 0.5倍。

修理: 即時（買い替え）。資金投入でその場で解消。

Wet系破損: 獲得データ 0.8倍。

修理: 1ターン必要。業者の手配と環境安定化のため、翌ターンまで解除されない。

3.3. 設備Tier

購入する設備ランクにより、データ量と成功率が変動する（不正倍率と乗算される）。

Tier 1: 1.0倍 / 補正無

Tier 2: 1.2倍 / 失敗率 -10%

Tier 3: 1.5倍 / 失敗率 -20%

Tier 4: 2.0倍 / 失敗率 -30%

4. ユニットシステム

4.1. パラメータ

Dex (器用さ): Wet実験成功率。

Comp (ITスキル): Dry解析成功率。

Diligence (真面目さ): サボり・器具破損の低減。

HP: 0になるとドロップアウト（失踪・退学）。

親愛度: 高いと上位課程への進学率アップ。

5. 研究・投稿ロジック

5.1. 研究ティアとハズレ

Tier 1: 基礎。5pt獲得。

Tier 2: 応用。10pt。ここで「当たり/ハズレ」が内部確定。

Tier 3: 核心。

当たり: 大量pt + 特殊データ獲得。Hard論文フラグ。

ハズレ: 成功判定でも「再現性なし」となり、データptが得られない泥沼。

5.2. 論文投稿

採択率は以下の式で算出される：


$$P_{accept} (\%) = \min(Base\% + \text{投資Data pt}, 99\%)$$

Easy: Base 60% / データptのみ

Middle: Base 40% / データptのみ

Hard: Base 20% / データpt + 特殊データ (Tier 3成功1回分相当)

6. 倫理と罰 (Ethics)

実験結果確認時、プレイヤーは「解釈を工夫」「外れ値除外」「捏造」などのQRPs（研究不正疑義）を選択できる。

メリット: 獲得データptの爆発的増加（最大2.5倍、捏造は失敗を20ptへ置換）。

リスク: 不正カウンターが蓄積。

Lv.1: 年次査察での即死判定。

Lv.2: 毎月の内部告発判定。

Lv.3: 確定バッドエンド（逮捕・追放）。

7. UI/UX 構成

メインラボ画面: 学生がわちゃわちゃ動くドット絵。タップで個別指示。

秘書室（栞里）: 収支計算書、各種設定、ヒントの提供。

演出: 論文採択時は華やかなエフェクト、不採択時は「Reviewer 2」の辛辣なコメントを表示。

8. 勝利・敗北条件

勝利: 10年完走。最終スコアによるランク付け。「真理（Nature級）」到達。

敗北:

資金ショート（3ヶ月連続赤字）。

研究室無人化（年度初めに学生0名、助教・Ph.D不在）。

逮捕（査察・告発による不正発覚）。

## 9. テーマ期待度（☆）システム（救済ではなく「事前契約」による納得設計）

本作では「研究テーマには当たり外れが内在する」ことをゲームの核とし、プレイヤーが**事前にリスクとリターンを理解した上で意思決定できる**ように、各研究テーマに「期待度（☆1〜☆3）」を付与する。  
これは失敗時の“救済”ではなく、**最初から条件を明示することで理不尽さを減らし、失敗を「判断の結果」として受け止められるようにする仕組み**である。

### 9.1 期待度（☆）の分類と基本性格

- **☆1：堅実テーマ（確実型）**
  - 到達上限：**Tier2止まり**
  - 成功率：**原則100%（研究成功判定は確定成功扱い）**
  - 特徴：小さく確実に成果が出る。スコア爆発はしないが、研究室の運営安定に寄与。
  - 想定：教育・基盤整備・再現性重視の研究。

- **☆2：応用・発展テーマ（中リスク型）**
  - 到達上限：**Tier3到達可能**
  - Tier3運用：**Tier3で実行できる実験量（試行回数）に上限**（例：月あたり/年度あたりの回数制限）
  - 特徴：安定と挑戦のバランス。勝ち筋として成立する標準枠。

- **☆3：実験的テーマ（高リスク型）**
  - 到達上限：**Tier3到達は確率**
  - 特徴：当たれば大きいが、外れれば長期停滞し得る。評価・資金・学生の圧力が強くなる。
  - 想定：野心的・探索的研究。プレイヤーに「賭け」を要求する枠。

### 9.2 Tier構造との対応（設計上の意図）

期待度（☆）は「Tier2到達時点で何が決まるのか」をプレイヤーに分かる形で提示する。

- ☆1：Tier2で成果は確定（ただしTier3は存在しない）
- ☆2：Tier3へ進めるが、Tier3での実験量に制約がある
- ☆3：Tier3到達が確率（当たり/外れの内在を、選択時点で合意させる）

この設計により、ハズレ研究による破綻を「理不尽」ではなく「選択の結果」として扱える。

### 9.3 UI/演出（納得性のための提示）

- テーマ選択画面で、期待度（☆）と短い説明タグを表示（例：堅実／王道／実験的・博打）
- ☆3選択時は警告演出（例：「探索研究：成果が出ない可能性があります」）を必ず挿入
- 期待度（☆）はテーマの“性格”であり、プレイヤーの能力値で☆そのものを改変できない（改変できると救済化するため）

### 9.4 研究不正の扱い（倫理は攻略手段にしない）

本作は研究者倫理を重視する方針とし、**研究不正は「効率化の攻略法」ではなく「論外の禁忌」**として扱う。

- 不正を選択した時点で **原則アウト（ゲームオーバー/強制終了）**
- 不正は「誘惑」として存在し得るが、成功・得点化しない
- 目的：プレイヤーに「追い詰められる状況」を提示しつつ、倫理の一線をゲーム的に明確化する
